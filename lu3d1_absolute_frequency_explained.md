# `176Lu+ (3D1)` 光频标绝对频率测量论文解读

本文档面向第一次接触光钟、频率标准和时间溯源的读者，整理自论文：

- 题目：Absolute frequency measurement of a Lu+ (3D1) optical frequency standard via link to international atomic time
- 作者：Zhao Zhang, Qi Zhao, Qin Qichen, N. Jayjong, M. D. K. Lee, K. J. Arnold, M. D. Barrett
- 版本：arXiv:2502.10004v3，2025-05-28
- 期刊信息：Metrologia 62, 035008 (2025)
- 本地文件：`2502.10004v3.pdf`
- 在线链接：https://arxiv.org/abs/2502.10004

## 1. 这篇文章一句话在做什么

作者第一次报告了 `176Lu+` 离子中 `1S0 -> 3D1` 光钟跃迁的绝对频率：

```text
353 638 794 073 800.35(33) Hz
```

括号里的 `(33)` 表示最后两位的不确定度，也就是约 `0.33 Hz`。相对到 `3.5e14 Hz` 量级的光频上，分数不确定度是：

```text
9.2e-16
```

直观理解：他们测的是一个每秒振荡约 `353.6 万亿次` 的“光学节拍”，最后把误差压到大约 `千万亿分之一` 的量级。

## 2. 为什么这件事重要

现在 SI 秒的定义来自铯原子 `133Cs` 的微波跃迁。铯喷泉钟已经非常好，但很多光钟能做到更低的不确定度。未来如果重新定义“秒”，很可能会依赖光钟。

不过，一个新的光钟体系要被国际计量体系正式采纳，不能只说明“它很稳定”。还必须回答一个基本问题：

```text
这个光钟的频率到底是多少，并且这个数能不能溯源到 SI 秒？
```

这篇论文的意义就在这里：

1. `176Lu+` 是一个有潜力的光钟离子体系。
2. 之前已经有人证明它可以做得很准。
3. 本文首次给出 `Lu+ (3D1)` 光频标的绝对频率。
4. 作者把本地实验室的光钟，通过氢钟、GPS、UTC(USNO)、TAI 和 BIPM 的 Circular T 报告，接到了 SI 秒。

如果后续有更多实验室复现并降低不确定度，`Lu+` 才有机会进入 CIPM 推荐频率列表，进一步成为“秒的次级表示”候选。

## 3. 先把几个核心概念讲清楚

### 3.1 光钟是什么

普通钟表靠摆轮、石英或电子振荡器计时。原子钟靠原子内部两个能级之间的固定跃迁频率计时。光钟使用的是光学频率跃迁，频率通常在 `1e14 Hz` 量级，比微波铯钟高很多。

频率越高，不代表天然更准，但高频给了更细的刻度。只要能把环境扰动控制住，光钟就能做到极高的相对精度。

### 3.2 `176Lu+` 是什么

`Lu` 是镥元素，`176Lu+` 表示质量数为 176、带一个正电荷的镥离子。实验中作者只困住一个离子，用它作为频率标准。

“单离子光钟”的直觉图像是：

```text
把一个离子关在电磁场形成的小陷阱里，
用激光冷却它，
再用很窄线宽的钟激光去问它：
你的某个能级跃迁频率是不是这个值？
```

如果激光频率偏低或偏高，离子的响应会变。实验系统根据响应不断修正激光频率，最后让激光锁到离子的跃迁频率上。

### 3.3 什么叫绝对频率

“绝对频率”不是只和另一个实验室的光钟比谁快谁慢，而是要给出以 SI 秒为单位的频率值，单位是 Hz。

本文要测的是：

```text
Lu+ 光钟频率 / SI 秒
```

也就是：如果用国际单位制定义的一秒来数，这个光学跃迁一秒振荡多少次。

### 3.4 为什么需要 TAI

实验室不可能直接拿着“理想 SI 秒”来比较。现实中，国际计量局 BIPM 维护国际原子时 TAI，并通过全球实验室的钟数据和校准报告，给出 TAI 与 SI 秒之间的关系。

所以作者的路线是：

```text
Lu+ 光钟
-> 本地氢钟 HM
-> GPS/PPP 链接到 UTC(USNO)
-> BIPM Circular T 链接到 TAI
-> Circular T 中 TAI 相对 SI 秒的校正
```

这条链路就是“溯源链”。

## 4. 论文的整体实验路线

可以把整篇文章拆成两条主线。

第一条是“本地光钟怎么工作”：

```text
单个 176Lu+ 离子
-> 激光冷却和态制备
-> 用 848 nm 钟激光和微波脉冲做询问
-> 用超精细平均得到 Lu+ (3D1) 标准频率
-> 用光频梳把光频和本地氢钟比较
```

第二条是“本地频率怎么接到 SI 秒”：

```text
本地氢钟
-> GPS 精密单点定位 PPP
-> UTC(USNO)
-> TAI
-> SI 秒
```

最后作者把所有频率偏移和不确定度合在一起，得到最终绝对频率。

## 5. Lu+ 光钟本身怎么定义

`176Lu+` 的基态是 `1S0`，本文关注的是到 `3D1` 态的光学跃迁。`3D1` 态有多个超精细能级，作者选了 `F = 6, 7, 8` 三条跃迁做平均。

光钟参考频率定义为：

```text
f_Lu = (nu_8 + nu_7 + nu_6) / 3
```

其中：

- `nu_8` 是从基态到 `3D1, F=8` 的跃迁频率。
- `nu_7` 是从基态到 `3D1, F=7` 的跃迁频率。
- `nu_6` 是从基态到 `3D1, F=6` 的跃迁频率。

为什么要平均？因为这样可以抵消很多对环境很敏感的项，例如二阶塞曼效应和电四极矩相关的张量频移。对小白来说，可以理解成：

```text
三条线各自会被环境拉偏，
但拉偏方向和大小有结构性关系。
把它们按设计好的方式平均，
很多扰动就互相抵消了。
```

实际实验中，作者不是分别测三条光学线再手工平均，而是用“光学 Ramsey 询问 + 微波脉冲转移”的组合，在一次询问序列中实现这个平均。论文给出的等效公式是：

```text
f_Lu = (nu_8 + nu_7 + nu_6) / 3
     = nu_L + (2 f_1 + f_2) / 3
```

这里：

- `nu_L` 是 848 nm 钟激光的频率。
- `f_1` 和 `f_2` 是实验中使用的两个微波频率。
- 只要脉冲时序安排正确，即使微波没有完全共振，上式仍然表示超精细平均后的钟频率。

## 6. Ramsey 和 hyper-Ramsey 方法在这里起什么作用

### 6.1 Ramsey 询问

Ramsey 方法可以理解成“先轻轻推一下原子，等一段时间，再推一下，然后看结果”。两次脉冲之间，原子相位会积累。如果激光频率刚好等于原子跃迁频率，相位积累的结果会有一个特征图样；如果激光偏了，图样会偏。

实验系统利用这个偏差信号来调激光，让激光锁到原子频率上。

本文使用的主要时序参数包括：

- 光学脉冲时间 `tau_L = 1.4 ms`
- 微波脉冲时间 `tau_1 = tau_2 = 2 ms`
- 总 Ramsey 询问时间 `T_R = 42 ms`
- 每 `25` 个循环更新一次激光伺服，约 `3.6 s`
- 询问占总周期约 `60%`

### 6.2 hyper-Ramsey 抑制光移

钟激光本身也会扰动原子，造成 ac Stark shift，也叫光移。hyper-Ramsey 的思路是通过额外的相位翻转和脉冲设计，让这种光移对最终锁定频率的影响大幅减小。

本文在序列末尾前加入一个相位移动 `180 deg` 的 848 nm `pi` 脉冲，用来抑制剩余的探测光诱导 ac Stark shift。

## 7. 光频怎么和氢钟比较

`Lu+` 光钟频率大约 `353 THz`，氢钟输出常见的是 `10 MHz`。这两个频率差了七八个数量级，不能直接用普通电子计数器比较。

中间要靠“光频梳”。光频梳可以想象成一把跨越光学频率和射频频率的尺子：

```text
光频梳的一端能和光学激光拍频，
另一端能和微波/射频参考相连。
```

本文的链路是：

1. 848 nm 钟激光被锁到 `176Lu+` 原子参考。
2. 商用掺铒光纤飞秒光频梳与钟激光拍频。
3. 光频梳重复频率约 `250 MHz`，被参考到 Lu+ 稳定的钟激光。
4. 光频梳的载波包络偏移频率 `f_o` 锁到本地氢钟。
5. 氢钟的 `10 MHz` 信号倍频到 `980 MHz`。
6. 它和光频梳重复频率的四次谐波附近信号混频。
7. 得到约 `20 MHz` 的下变频信号。
8. 用 SDR 零死时间连续测相位。

论文报告，在光钟运行期间没有观察到光频梳 cycle slip。也就是说，频率梳没有跳错齿，这对绝对频率测量非常关键。

## 8. 为什么还需要氢钟当“飞轮”

光钟非常准，但实验上不能一直运行。本文的 Lu+ 光钟在三个月中间歇运行，总运行时间是 `162 h`，平均每段连续运行约 `2.9 h`。

TAI 和 Circular T 的数据以 5 天和 1 个月为时间窗口。光钟只在其中一部分时间开着，其余时间怎么办？

答案是用氢钟 HM 当飞轮。

飞轮的直觉是：

```text
光钟开着的时候，用光钟校准氢钟；
光钟停着的时候，假设氢钟按自己的噪声模型继续走；
最后估计这个“补空档”的不确定度。
```

这就是论文中很重要的 dead time interpolation，也就是“死时间插值”问题。

## 9. 远程接到 SI 秒的链路

作者所在实验室没有自己的 UTC(k) 本地实现，所以他们选择远程连接到 `UTC(USNO)`，也就是美国海军天文台维护的 UTC 本地实现。

具体步骤是：

1. 本地 GNSS 接收机由本地氢钟参考。
2. 远程 USN7 GPS 接收机由 UTC(USNO) 参考。
3. 两边的 RINEX 数据用于 PPP 处理。
4. 用 Bernese GNSS 软件每 `30 s` 估计一次本地 HM 与 UTC(USNO) 的相位差。
5. BIPM 的 Circular T 报告给出 UTC(USNO) 到 UTC/TAI 的关系。
6. Circular T 还给出 TAI 标度间隔相对 SI 秒的偏差。

论文把完整频率链写成：

```text
f_Lu / f_SI
= (f_Lu,T1 / f_HM,T1)
* (f_HM,T1 / f_HM,T2)
* (f_HM,T2 / f_UTC(USNO),T2)
* (f_UTC(USNO),T2 / f_TAI,T2)
* (f_TAI,T2 / f_TAI,T3)
* (f_TAI,T3 / f_SI,T3)
```

其中：

- `T1` 是 Lu+ 光钟实际运行的时间段。
- `T2` 是和 Circular T 对齐的 5 天窗口。
- `T3` 是 Circular T 的约 1 个月窗口。
- `f_HM,T1 / f_HM,T2` 表示氢钟在光钟停机时间中的插值问题。
- `f_TAI,T2 / f_TAI,T3` 表示 TAI 在 5 天窗口到月度窗口之间的插值问题。

这个公式看起来长，但意思很简单：

```text
每一段都是一个可测或可查的比值。
把所有比值乘起来，就把 Lu+ 光钟接到了 SI 秒。
```

## 10. 主要频移和系统误差

光钟不是放在那里就天然等于理想原子频率。磁场、电场、黑体辐射、运动、重力势都会让频率偏移。作者逐项估计这些偏移并给出不确定度。

论文表 1 的单位是 `1e-18` 的分数频率。主要结果如下：

| 效应 | 频移 | 不确定度 | 小白解释 |
| --- | ---: | ---: | --- |
| 二阶塞曼频移 | `-140.7` | `0.6` | 磁场造成的二阶频率偏移 |
| 过剩微运动 | `-0.2` | `0.2` | 离子在 Paul 阱里残余受驱运动导致的相对论效应 |
| 二阶多普勒 | `-0.1` | `0.1` | 离子热运动导致的时间膨胀 |
| 射频 ac-Zeeman | `0.2` | `<0.1` | 阱驱动射频磁场造成的影响 |
| 微波 ac-Zeeman | `-8.0` | `6.2` | 微波极化不平衡带来的磁场相关频移 |
| 微波耦合误差 | `0.0` | `3.7` | 微波脉冲不完美导致超精细平均不完美 |
| 848 nm ac-Stark | `0.0` | `1.2` | 钟激光探测本身造成的光移 |
| 黑体辐射 | `-1.6` | `0.3` | 环境热辐射改变能级 |
| 小计 | `-150.4` | `7.3` | Lu+ 装置本身的频移 |
| 引力红移 | `1718` | `36` | 实验室高度相对大地水准面的频率修正 |
| 总计 | `1568` | `37` | 加上引力后的总修正 |

注意这里的 `1718` 是 `1718e-18 = 1.718e-15`。引力红移是最大的修正项，因为实验室不在理想大地水准面上。

### 10.1 二阶塞曼频移

即使做了超精细平均，仍会残留二阶塞曼频移。作者使用已知系数：

```text
-4.89264(88) Hz/mT^2
```

实验期间测得平均磁场约：

```text
0.10084(20) mT
```

对应频移：

```text
-49.75(20) mHz
```

换成分数频率就是约：

```text
-1.407e-16
```

### 10.2 引力红移

根据广义相对论，位置越高，钟走得越快一点。近地面近似公式是：

```text
Delta f / f = g h / c^2
```

作者测量并估计：

- 本地重力加速度 `g = 9.7803(3) m/s^2`
- 离子相对 WGS84 椭球的高度 `H = 23.68(10) m`
- 大地水准面起伏 `N = 7.89(10) m`
- 用 `h = H - N` 得到相对大地水准面的高度

最终引力红移修正约：

```text
1.72(4)e-15
```

这在最终频率中必须修正。

## 11. 统计分析和死时间插值

测量发生在 2021 年 3 月到 5 月，MJD `59270` 到 `59338`。作者把数据按 Circular T 的 5 天窗口分段。

每个 5 天窗口中，Lu+ 光钟的运行占比并不一样：

```text
11%, 2%, 7%, 37%, 14%, 9%, 3%, 30%, 20%
```

运行占比越低，氢钟需要“代替光钟走”的时间越长，死时间插值不确定度越大。例如论文表 2 中：

- 光钟只运行 `2%` 的窗口，HM 插值不确定度达到 `29e-16`。
- 光钟运行 `37%` 的窗口，HM 插值不确定度降低到 `6.8e-16`。
- 光钟运行 `30%` 的窗口，HM 插值不确定度约 `7.6e-16`。

### 11.1 作者如何估计 HM 插值不确定度

他们不是简单拍脑袋给误差，而是建立氢钟噪声模型并做数值模拟。

模型包含：

- 白相位噪声
- 白频率噪声
- 闪烁频率噪声
- 一个用于拟合 5 到 60 分钟不稳定度平台的额外低通白频率噪声项

然后作者用 FFT 方法生成约 `2000` 条模拟氢钟轨迹。对每条轨迹，比较：

```text
只在 Lu+ 实际开机时间 T1 上取平均
```

和：

```text
在完整 5 天窗口 T2 上取平均
```

两者差值的标准差，就是死时间插值不确定度。

这个方法的直觉是：

```text
假装氢钟按已知噪声规律走很多次，
看看“只抽样光钟开机时刻”会和“完整五天平均”差多少。
```

## 12. 最终不确定度预算

论文最终把三个月 Circular T 报告 399、400、401 合并。表 3 给出的总不确定度单位是 `1e-16`。

| 来源 | 最终贡献 |
| --- | ---: |
| Lu/TAI 统计不确定度 | `7.3` |
| TAI 插值 | `1.5` |
| TAI/SI | `0.4` |
| 总统计不确定度 | `7.4` |
| HM 漂移模型 | `2.0` |
| HM 昼夜相关效应 | `5.0` |
| Lu+ 系统误差 | `0.1` |
| 引力红移 | `0.4` |
| 主/次级频率标准 PSFS | `0.9` |
| 总系统不确定度 | `5.5` |
| 总不确定度 | `9.2` |

最终的 `9.2e-16` 主要不是 Lu+ 光钟本身限制的。真正限制来自：

1. 卫星时间传递链路的稳定性。
2. 光钟不是连续运行，需要氢钟在中间当飞轮。
3. 可能存在的日周期环境相关偏差。

这一点很重要：本文不是说 Lu+ 光钟本身只能做到 `1e-15`。相反，Lu+ 系统误差在总预算中很小，主要瓶颈在“如何把本地光钟接到国际时间尺度”。

## 13. 最终结果怎么得到

作者先在每个 5 天窗口得到 `f_Lu / f_TAI`，再按 Circular T 的月度窗口求加权平均，最后通过 TAI 到 SI 秒的校正得到绝对频率。

三个月结果的加权平均是：

```text
f_Lu = 353 638 794 073 800.35(33) Hz
```

分数不确定度：

```text
9.2e-16
```

论文还报告三个整月结果之间的一致性较好，约化卡方 `chi^2_nu = 0.56`，自由度为 2。这说明三个独立月度结果没有显示明显互相矛盾。

## 14. 作者认为下一步怎么改进

论文讨论了几个改进方向：

1. 用 PPP-AR 改善卫星时间传递稳定性。
2. 移动 GNSS 天线位置，减少附近建筑遮挡天空造成的问题。
3. 提高 Lu+ 光钟运行占空比。作者提到报告准备期间已经实现连续 10 天约 `95%` 的开机率。
4. 改善 350 nm repump 光源，减少 701 nm 激光 mode hopping 引起的中断。
5. 通过稳定光纤链路连接新加坡国家计量中心 NMC。
6. 使用更稳定的氢钟或氢钟组合作为更好的本地飞轮。

作者认为，经过这些改进，未来做到低 `1e-16` 量级的绝对频率不确定度是可行的。

## 15. 这篇文章的方法论总结

如果只看方法，本文的套路可以抽象成下面几步：

1. 选一个原子或离子的窄线宽光学跃迁作为频率标准。
2. 设计询问序列，把环境敏感性尽量抵消或压低。
3. 用激光伺服把钟激光锁到原子跃迁上。
4. 用光频梳把光学频率变成可和微波参考比较的频率。
5. 用本地高稳定微波钟当飞轮。
6. 用卫星时间传递或光纤链路把本地飞轮接到国际时间尺度。
7. 用 BIPM 报告把 TAI 接到 SI 秒。
8. 对每个环节列出频移修正和不确定度。
9. 用加权平均和不确定度合成得到最终绝对频率。

这套方法不只适用于 Lu+，也是很多光钟绝对频率测量的基本框架。

## 16. 小白读完应该抓住的重点

最核心的不是“他们测了一个很大的数字”，而是他们证明了这个数字可以通过完整链路接到 SI 秒。

本文的关键贡献可以概括为：

- 首次给出 `Lu+ (3D1)` 光频标的绝对频率。
- 使用单个 `176Lu+` 离子，借助超精细平均降低环境敏感性。
- 用光频梳把 `353 THz` 光频和本地 `10 MHz` 氢钟连接起来。
- 用 GPS/PPP、UTC(USNO)、TAI 和 Circular T 把实验室频率接到 SI 秒。
- 最终不确定度 `9.2e-16`，主要受时间传递和氢钟插值限制，而不是 Lu+ 光钟本身限制。

## 17. 术语速查

| 术语 | 含义 |
| --- | --- |
| SI 秒 | 国际单位制中“秒”的定义单位，目前基于铯原子跃迁 |
| 光钟 | 使用光学频率跃迁作为参考的原子钟 |
| 绝对频率 | 以 SI 秒为单位给出的频率值 |
| `176Lu+` | 质量数 176 的单电荷镥离子 |
| `1S0 -> 3D1` | 本文使用的 Lu+ 光学钟跃迁 |
| 超精细平均 | 对多个超精细跃迁做设计好的平均，以抵消部分系统频移 |
| Ramsey 询问 | 用两次相干脉冲和中间自由演化测频率偏差的方法 |
| hyper-Ramsey | 改进的 Ramsey 序列，用来抑制探测光引起的光移 |
| 光频梳 | 连接光学频率和射频/微波频率的“频率尺” |
| 氢钟 HM | 高稳定微波参考，在本文中作为本地飞轮 |
| 飞轮 | 主钟停机时暂时代替它保持时间/频率连续性的参考 |
| UTC(k) | 某个实验室实现的 UTC 本地版本 |
| UTC(USNO) | 美国海军天文台实现的 UTC 本地版本 |
| TAI | 国际原子时，由 BIPM 维护 |
| Circular T | BIPM 每月发布的时间尺度比较和校准报告 |
| PPP | 精密单点定位，用 GNSS 数据做高精度时间/相位比较 |
| 死时间插值 | 光钟未运行时，用飞轮补齐时间窗口并估计额外不确定度 |
| 分数不确定度 | 绝对不确定度除以频率本身，例如 `0.33 Hz / 3.536e14 Hz` |
