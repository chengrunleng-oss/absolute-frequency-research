# `lu3d1_absolute_frequency_explained.md` 小白复现视角审核反馈

审核对象：`lu3d1_absolute_frequency_explained.md`

对照材料：`2502.10004v3.pdf`

审核视角：假设读者第一次接触光钟、频率标准、TAI、GNSS 时间传递和不确定度预算，但目标不是只读懂大意，而是要尽可能复现论文中的工作或至少复现论文的数值分析。

## 总体结论

当前解读文档适合作为“论文科普导读”，能帮助小白知道论文大概在做什么、为什么重要、最终结果是多少。但它还不是“复现指南”。如果小白按当前文档尝试复现，会很快卡在三个地方：

1. 不知道“复现”到底是复现实验硬件、复现数据分析，还是只复算论文表格和最终频率。
2. 不知道需要哪些原始数据、这些数据从哪里下载、哪些数据论文没有公开。
3. 不知道如何把公式、时间窗口、Circular T、PPP、氢钟模型、不确定度合成一步步算成最终的 `353 638 794 073 800.35(33) Hz`。

建议把现有文档定位为“概念版解读”，另外补一份“复现版操作手册”。下面反馈按阻断程度分级。

## P0：直接阻断复现的问题

### 1. 没有定义复现目标和可复现边界

文档开头说面向第一次接触的读者，但没有说明读者最后要复现到什么程度。对小白来说，“复现这篇文章”至少有三种完全不同的含义：

- 完整实验复现：搭建单离子光钟、激光系统、Paul 阱、光频梳、氢钟、GNSS 链路。
- 数据分析复现：使用作者已有测量数据，复算频率链、不确定度预算和最终结果。
- 公开数据复现：只用论文、Circular T、CDDIS RINEX 等公开数据，尽量复算能公开复算的部分。

这三种目标的难度、数据需求、设备需求完全不同。当前文档没有告诉小白哪一种可行，容易让读者误以为只靠文档和 PDF 就能完整复现。

建议补充：

- 一张“复现层级表”，说明每个层级能复现什么、不能复现什么。
- 明确指出作者的本地 Lu+/HM 原始数据并未随论文公开，论文 Data availability 写的是可向通讯作者合理请求。
- 明确告诉读者：没有作者原始数据时，无法独立复算全部最终数值，只能复现流程、公开链路数据处理和部分表格计算。

### 4. 缺少可运行的分析环境说明

### 5. 频率链公式缺少符号、正负号和单位解释

```text

```

但小白无法仅凭这个公式计算，因为很多符号没有定义到可计算程度：

- “假设氢钟按自己的噪声模型继续走”应补充：不是直接补一个确定值，而是估计采样时间不完整带来的平均频率不确定度。
- “没有观察到 cycle slip”应说明检查方法或至少说明这是实验数据质量控制结论。

### 21. 没有区分“论文已有结论”和“解读文档推断”

有些地方是论文直接给出的，有些地方是文档作者的解释。小白复现时需要知道哪些是可引用事实，哪些是辅助理解。

建议在关键数值旁标注：

- 论文直接报告。
- 由论文表格换算。
- 解读文档为便于理解的类比。
- 需要查参考文献 [8]、[13]、[25] 等才能复现。
